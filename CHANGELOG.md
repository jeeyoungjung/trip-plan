# Changelog

- 2026-05-24: Initial site files added.
- 2026-05-23: Stop 14 (Sat afternoon) fleshed out with three options as sub-points: Chinatown→LES→Union Sq, Tribeca+SoHo loop, Whitney+Meatpacking. Stop stays `tbd` so it renders dashed in the sidebar with no main map marker; sub-points let the map show the three rough anchor areas.
- 2026-05-23: Day-of weather pill in each day-section header via Open-Meteo (no key, no deps). Silent fallback if fetch fails or trip dates fall outside the 16-day forecast horizon.
- 2026-05-23: Floating Ask chat panel (bottom-right of map) wired to Anthropic Messages API with claude-haiku-4-5. User pastes their own key — held in memory only, never persisted (per the no-storage rule). System prompt embeds the itinerary JSON so answers are grounded in the trip. Three quick-prompt chips for common questions.
- 2026-05-23: Chat backend switchable via `config.local.js` (gitignored). Default uses Azure AI Foundry Responses API at `…/openai/v1/responses` with deployment `gpt-chat-latest`. When the config file is present the key-paste UI is hidden; when absent (e.g., GitHub Pages) the chat falls back to the Anthropic key-paste flow. Response extraction handles both `output_text` and the `output[].content[].text` walk.
- 2026-05-23: Chat now invokes the Foundry Agent `jeeyoungbot` (v2) via `agent_reference` on the project-scoped endpoint `…/api/projects/travel-planning-bot/openai/v1/responses` — the agent has its own Korean-default instructions and web_search tool. We still prepend the current itinerary JSON as a seed message so the agent grounds replies in the actual plan. The welcome message shows the active agent name.
- 2026-05-23: Chat UI redesigned for editorial fit. Floating pill button bottom-right with eyebrow "Ask" + serif italic "Jeeyoung-bot" and a terracotta pulse. Desktop: right-side drawer (440px) slides in with a blurred backdrop dim; click backdrop or press Esc to close. Mobile (≤900px): full-viewport takeover sliding from bottom, no backdrop, body scroll locked while open. Internal styling moved from "messenger bubbles" to typographic indent lines — user messages right-aligned with an ink rule, assistant messages with a JB monogram and day-2 gray rule. Suggestions chipped under a mono "Try asking" label. Chat DOM moved out of `.map-wrap` so it can be truly fixed.
- 2026-05-23: Serverless proxy at `api/chat.js` so the Azure key lives server-side and visitors don't need to paste a key. Browser defaults to `POST /api/chat`; the function pins `agent_reference` to `jeeyoungbot` v2, attaches `AZURE_KEY` from env, and enforces optional `ALLOWED_ORIGIN`. `config.local.js` remains as a LOCAL-DEV-ONLY override that calls Azure directly. `vercel.json` sets the function max duration to 30s. Deploy instructions added to README.
- 2026-05-23: Chat polish + mobile keyboard fix. Panel height now driven by a `--chat-h` CSS var; JS syncs it to `window.visualViewport.height` while open on narrow viewports so the composer stays visible when the soft keyboard appears. Composer + suggestions get `flex-shrink: 0` so they don't squeeze. `safe-area-inset-*` envs added so the Ask button and form don't sit under the iPhone home indicator. Bumped chat z-index to 999997–999999 to sit above any third-party widgets. Backdrop blur raised from 2px → 6px with a touch of saturation. User-message hard right-align dropped in favor of a subtle ink-on-cream tint; "JB" monogram repositioned. Suggestions and inputs got refined focus states + crisper hit areas.
- 2026-05-23: Floating button renamed from "Ask · Jeeyoung-bot" (two-line) to a single-line "🤖 Jeeyoung" with the pulse moved to the right side. Bot emoji tilts on hover. Textarea and Send button locked to exactly 44px so they align flush at the form bottom. Placeholder simplified to "Ask Jeeyoung…".
- 2026-05-23: Two chat bugs fixed.
  - **Error 405 on send.** Proxy switched from CommonJS to `export default` (with `package.json {"type":"module"}`) so Vercel routes the function reliably; added a `GET /api/chat` health check; client now surfaces a specific Korean message when a 405 comes from `python3 -m http.server` (which only allows GET/HEAD) directing the user to use `config.local.js` or the deployed URL instead.
  - **Panel disappears when mobile keyboard opens.** Body lock now uses `position: fixed; top: -savedScrollY` (with scroll restore on close) so iOS Safari can't scroll behind the panel. Panel `top` is also driven by `visualViewport.offsetTop`, so any iOS-induced offset shift is followed precisely instead of dragging the panel off-screen. Focus is blurred before close so the keyboard collapses before scroll restores.
